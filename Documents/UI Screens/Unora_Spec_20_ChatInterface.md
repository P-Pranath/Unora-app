## 2. Screen Purpose & Context

### 2.1 The Reward

This is the **reward screen** â€” the culmination of 15 days of consistent effort. The chat interface is where anonymous connection transforms into real conversation.

**Philosophy:**
> "15 days of showing up. Now, the conversation begins."

This screen must feel:

| Quality | Implementation |
|---------|----------------|
| **Earned** | Visual celebration of the Day 15 achievement |
| **Premium** | Dark, cinematic aesthetic â€” not a generic white messenger |
| **Intimate** | Focus on the conversation, minimal distractions |
| **Safe** | Clear access to safety controls (report/unmatch) |
| **Warm** | Gold accents and server colors create emotional warmth |

### 2.2 Identity State

The partner is **no longer anonymous**. This screen displays:

| Data | Display Location |
|------|------------------|
| **Real Name** | Header (primary text) |
| **Face Photo** | Header avatar (48px, clear, unblurred) |
| **Day 15 Badge** | Sub-header indicator |

### 2.3 Critical Constraints

> [!CAUTION]
> These rules are **non-negotiable**.

| Constraint | Rule |
|------------|------|
| **Pre-requisite** | Chat is ONLY accessible after Day 15 completion |
| **Text Only** | No video calls, voice rooms, or media sharing (not in PRD) |
| **Real Identity** | Header must show verified name and real photo |
| **Safety Controls** | Report/Unmatch must be accessible in header menu |
| **No Anonymity** | Both users see each other's real identity |

## 13. Technical Notes

### 13.1 API Requirements

```typescript
interface ChatMessage {
  id: string;
  connectionId: string;
  senderId: string;
  content: string;
  timestamp: string;           // ISO timestamp
  status: 'sending' | 'sent' | 'delivered' | 'read';
}

interface ChatPartner {
  userId: string;
  name: string;                // Full legal name
  firstName: string;           // For personalization
  photoUrl: string;            // Face photo
  connectionId: string;
}

interface SendMessageRequest {
  connectionId: string;
  content: string;
}

interface TypingIndicator {
  connectionId: string;
  userId: string;
  isTyping: boolean;
}
```

### 13.2 Real-Time Requirements

| Feature | Implementation |
|---------|----------------|
| **Message Delivery** | WebSocket connection for real-time |
| **Typing Indicator** | Debounced (300ms delay before sending) |
| **Read Receipts** | Sent when message enters viewport |
| **Offline Queue** | Messages queued locally if offline |
